Computer network models

➙ Craig n Dave video client server and peer to peer networks

➙ TeachIct page about types of netorks

Points to remember:

  • A client-server network is a network which uses a central server.
  • In a client-server network, a client requests a service from a server.
  • In a peer-to-peer network, each device acts as the client and the server at the same time.
  • Advantage of client server networks: they are centralised, which makes security easier to manage for instance.
  • A server is a computer configured to wait for requests from clients and execute them. For instance, a web server stores webpages. It waits for clients request to sent back the requested webpages.
  • Types of servers include: web server (stores webpages), file server (stores files centrally), email server, dhcp server (gives IP addresses to the clients)...


Communication protocols and network metrics

➙ Craig n Dave video comparing wired and wireless

➙ Slides here

Points to remember:

  • Wi-Fi and 4G consume the most energy but have the largest range
  • Zigbee is very simple to implement and low power: well suited to IoT devices
  • Infrared communication needed a clear line of sight (doesn't go through obstacles or walls)
  • Advantages of wired connection (ethernet): more stable/reliable as it is less prone to interferences, faster transmission as the bandwidth is larger than wireless connections, more secure as data can't be intercepted.
  • Advantages of wireless connections: convenience (you can move around), simpler to add mobile devices.


TCP IP protocol

➙ Introductory video about TCP IP

➙ Craig n Dave video about the TCP IP model

Points to remember:

  • When data is sent out, the TCP protocol is in charge of splitting the data into packets and labelling those packets
  • When data is received, TCP protocol is in charge of checking that all the packets have been received, (error control)
  • IP in charge of choosing the shortest path available to transmit the data packets


OSI model

➙ TeachICT webpage about OSI model

➙ Video of a real life example of how data travels through the OSI model

Points to remember:

  • The OSI model is used to describe data being sent between two computers on a network
  • The OSI model is made of 7 layers that must be remembered.
  • When data is sent from a computer to another one, it first travels down through all the layers (from application to physical), then via the cables to the recieving end where it goes up through all the layers (from physical to application)
  • Each time data is passed to another layer, a header or a trailer is added to the actual data with information like the IP address, MAC address etc. The data plus the new header or trailer is packaged up and transmitted to the next layer. This process is called encapsulation.


Transmission media

Points to remember:

  • Twisted pair cable is affordable but has a limited bandwidth, range and is more prone to interference compared to fiber optics
  • Fiber optics transmits data in the form of ligth pulses along a glass fibre. It is best used for long-distance transmission and when large bandwidth is needed.


Network hardware

➙ Craig n Dave video here

➙ TeachIct page about network components

Points to remember:

  • At home, the internet box comprises the modem, the router and a switch
  • The end devices are connected to a switch first, itself connected to a router then to a modem which allows connection to the internet
  • A switch receives data on its input ports and forwards it to the correct output port (and destination device)
  • A hub receives data on its input ports and forwards it to all the other port (and devices connected to them)
  • A router reads the IP address on data packets and forwards it to the correct LAN


IP and MAC addressing

➙ Craig n Dave video here

➙ Teach ICT page on addressing

Points to remember:

  • The MAC address is found on each computer's NIC (Network Interface Card). It is given by the manufacturer and never changes
  • An IP address is a unique identifier given to a computer as soon as it connects to a network.
  • A MAC address can be used to forward data packets to the correct device within a LAN, and to filter out incoming data (the router has a white list of mac addresses that are allowed to connect)
  • An IP address is used to identify every device connected to a same network


Network security

➙ Craig n Dave videos about threats and prevention

➙ Teach ICT page on threats

Points to remember:

  • Threats include: Fishing (social engineering), malware, SQL injections, DDOS attacks ...
  • Prevention methods include: installing and updating an antimalware software, firewall, education/prevention
  • Hardware firewalls are installed on the router or on a separate dedicated device
  • Software firewall is installer on the client's computer

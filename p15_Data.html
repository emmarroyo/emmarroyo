What you need to know

5.1 Data and information

5.1.1 Understand the difference between data and information.

5.1.2 Understand sources of and the difference between structured and unstructured data.

5.1.3 Understand the value to organisations of extracting meaningful information from data.

5.2 Structured data

5.2.1 Understand why databases are used to structure data

5.2.2 Understand the structure of a relational database

  1. tables
  2. primary keys
  3. foreign keys
  4. composite keys
  5. records

5.2.3 Understand the concept of entities and the relationships between them:

  1. one to one
  2. one to many
  3. many to many

5.2.4 Be able to interpret and create entity relationship diagrams for a given scenario.


5.3 Structured query language (SQL)

5.3.1 Understand how and why SQL is used to manipulate data and data structures

5.3.2 Know how to select and use appropriate SQL commands, features and functions to manipulate data:

  1. perform queries and subqueries
  2. create tables using appropriate data types
  3. populate tables/insert, amend, delete
  4. link tables (UNION, JOIN)
  5. use wildcards (% and _)
  6. group, order, count

Points to remember:

  • Data is strings, numbers without meaning
  • Information is data in context, it has meaning
  • Structured data is data which can be organised in a database / table (numbers, characters, dates ...)
  • Unstructured data is everything else: it is 80% of the data collected by companies (emails, photos, videos, posts...)
  • Companies collect data to help them make informed decisions, improve their customer service or to create personalised marketing campaigns

Points to remember:

  • Relational database help reduce redundancies hence reduce the storage space needed (compared with flat file database)
  • In a relational database, data is organised in tables made of fields (columns) and records (rows)
  • The primary key uniquely identifies each record
  • Foreign key create the relationships between the tables. A foreign key is a primary key from another table.


Entity relationship diagrams

➙ Introductory (Scottish) video about ERD

➙ TeachICT lesson

Points to remember:

  • An ERD is a diagram showing a database structure. It is made of entities (tables), attributes (fields) and relationships.
  • There are three different types of relationships: one to one, one to many, many to many
  • When designing an ERD, start by listing the main entities with some attributes
  • To find the relationship between two entities, for instance students and classes, ask yourself the question: ONE student is linked to ONE or MANY classes? ONE class is linked to ONE or MANY students?
